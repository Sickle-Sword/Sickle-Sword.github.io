ディレクトリ周りについて

　論文一本とか集中的に分析しているとファイルがごちゃごちゃになってしまう。とりあえずRstudioのProject機能を使うのは前提にして，そこから先どうディレクトリを整理していくかをちゃんとまとめているものは少ない。

基本的な方針

　プロジェクトファイル（.Rproj）が入っているディレクトリ（ここではaとする）がある。普通にやっているとこの中にスクリプトや分析元のデータ，中間データやアウトプット用のcsv，Rmarkdownのcacheとかが溜まっていく。
（イメージ）

a/
  ┣ hogehoge.Rproj
  ┣ script.R
  ┣ cache/
  ┣ raw_data.csv
  ┣ middle_data.csv
  ┣ reg_table1.csv
  ...

　分析結果が少ないうちはこれでもよかろうが，アレコレやっているうちに地獄と化す。ある程度ファイルの種類・用途でフォルダに分けていったほうが賢い。最低限データとスクリプトを分けるくらいはした方がいいだろう。
　

a/
  ┣ hogehoge.Rproj
  ┣ 01_script/
  ┃  ┣ script.R
  ┃  ...
  ┣ 02_input/
  ┃  ┣ raw_data.csv
  ┃  ...
  ┣ 03_middle/
  ┃  ┣ middle_data.csv
  ┃  ...
  ┣ 04_output/
  ┃  ┣ reg_table1.csv
  ┃  ...
  ...

Rmarkdownファイル（.Rmd）の扱い

　通常のRscriptは上記のような方法で管理していけば良い。ただし，Rmarkdownのファイルは，ディレクトリの扱いが特殊であるため注意が必要である。具体的には（1）ワーキングディレクトリの取り扱い，（2）生成物の誘導について書いていく。

正規表現の仕様変更（R 4.0.0～）

調べても意外と情報が得られなかったので一応メモ

従来の仕様

例えば

a <- 'aabb ;) 334'

のような表現にマッチさせるには，

stringr::str_extract(a, 'aabb ;) 334')

これではだめで，

stringr::str_extract(a, 'aabb ;\\) 334')
## [1] "aabb ;) 334"

のように正規表現を書かなければならない

'()'や'.'といった正規表現中で特別な意味を持つ文字をもともとの形でマッチさせるには，文字列の前にエスケープ用のバックスラッシュ'\'をつける必要があるが，'\'自体も特殊な文字扱いなので，最終的に'\\.'のような形になる。

バックスラッシュ2つ'\\'にマッチさせるなら，正規表現は'\\\\'になる。これは面倒。

新しい仕様

pythonなどで実装されているraw stringという正規表現の書き方が使えるようになった。r'...'のように書くと，'...'の中でバックスラッシュが特別扱いされなくなる。
すなわち，以下のコードでよくなる。

stringr::str_extract(string = a, pattern = r'(aabb ;\) 334)')
## [1] "aabb ;) 334"

ここで注意すべきは，特別扱いがなくなるのはバックスラッシュのみなので，')'にマッチさせるにはその前にバックスラッシュを置く必要がある点。

ちなみに，pythonだとr'...'でよいが，Rだとr'(...)'というように正規表現全体を括弧で括る必要がある。なんか新しい問題が起こりそうで恐い。

# python
import re

m = re.match(pattern = r'aabb ;\) 334', string = r.a)
m.group()
## 'aabb ;) 334'
